More About Lassen Peak Data Internship

May 2024-Sept 2024

During my internship, I developed diagnostic and performance benchmarking tools using Python and C++ to evaluate radar-based embedded systems. I worked closely with low-level hardware interfaces, testing system communication over I2C and SPI buses. To simulate real-world embedded constraints, I wrote custom scripts that monitored CPU and RAM usage in real time. I also built dynamic dashboards in Tableau and Excel (using advanced formulas and macros) to visualize trends in signal integrity and system performance. As part of the QA process, I contributed to debugging multi-threaded sensor data pipelines and worked within bug tracking systems to support real-time issue resolution. I practiced test-driven development using NUnit and wrote unit tests to help ensure the reliability of our embedded analytics workflows. I regularly used Microsoft Excel and PowerPoint to communicate findings and present technical results to cross-functional teams.

More About my Augmented Reality UX Designer Design Lead Role at the SCU Human Computer Interaction Lab

June 2024-present

I’m currently collaborating in an Agile team to build an Augmented Reality (AR) tour app designed to restore and share the voices of the Ohlone Native American people. This project has been recognized globally, winning three awards for its innovation and cultural impact. Using Unity and C#, I developed interactive game stops and helped implement the navigation system that guides users through the experience. As the design technical lead, I manage a team of seven SCU students—leading design meetings, assigning tasks, and resolving conflicts to keep the project on track. I’ve created onboarding screens, branding elements, and UI/UX flows using Figma, and worked on integrating authentic tribal elements through advanced 3D modeling in Blender and Unity.

More About my Lab Game Data Scientist Role at the Santa Clara University VR/AR Imaginarium

January 2024- June 2024

I built an eye-tracking analytics system in C, designed to operate on resource-limited platforms by extracting and processing behavioral data from Unity-based games. I structured game logs and transformed them into embedded-friendly CSV pipelines for further analysis. To ensure efficient real-time processing, I analyzed system latency and optimized the spatial pattern detection logic. I also developed a modular data processing pipeline using Python and SQL, tailored for compatibility with embedded environments. Working with multi-threaded analysis and live data streaming, I helped enhance both sensor responsiveness and overall system usability. As part of a research team, I also contributed to simulating RTOS scheduling models and refining device logic under tight hardware constraints.
